### 2018/10 - 現在 : 株式会社パネイル

職務: Web アプリケーションエンジニア

#### 電気小売事業管理システムの開発・運用

**使用技術スタック**

* フロントサイド: jQuery
* サーバーサイド: Python3.6, Django3.0
* インフラ: AWS(EC2, ALB, RDS, CloudWatch)

主にサーバーサイドの開発・運用を担当。

担当した機能は複雑な申込み日バリデーションの実装や連携システムからデータ取得処理、契約書郵送処理のなど。

電力業界の複雑な仕様をシステムに落とし込むため、Biz側と積極的に会話することを心がけた。
また、バグ報告や相談については担当外であったも積極的に対応した。

後に電気以外の新たな商材を扱う新機能の設計/実装を任される。
企画から運用までビジネスサイドと幅広く意見交換し、要件定義から画面設計まで行った。
最終的にはプロジェクトの状況変化により実装見送りとなった。

#### ESG 情報レポート作成システムの新規開発

**使用技術スタック**

* フロントサイド: React16.13, TypeScript3.9
* サーバーサイド: Python3.8, Django REST Framework3.11
* インフラ: Terraform0.12, AWS(ECS Fargate, ALB, RDS, CloudFront), Github Pages, Datadog

**前期: インフラ担当**

ゼロベースからインフラ設計・構築を担当。
環境の差異を少なくするためにdockerを採用。本番環境にはAWS ECS Fargateを使用した。
スピードが求められていたため、可能な限りシンプルな構成を選択。
一方で、変更管理を容易にするためTerraformを導入、運用基盤を構築した。

監視の観点では、APMによる開発効率化を魅力的に感じDatadogの導入を提案。
提案にあたってランニングコストの観点なども考慮し、承認後は構築まで実施した。

担当外ではプロジェクトの技術再選定を提案。
具体的には、サーバーサイドの開発言語をGoからPythonに変更して作り直そうというもの。
社内的に重要なプロジェクトであったので、社内に開発ノウハウがないGoの採用は品質面でも効率面でもリスクが高いという背景だった。
この提案はチーム内の検討を経て採用された。

**中期: PM補佐/スクラム導入支援**

一通りのインフラ構築を終え、プロジェクト全体の状況を見てフロントエンドチームの人手が足りなかったためマネジメント補佐に立候補。タスク整理や進捗状況の管理などを行った。

その中でbiz/dev間の距離を感じ、上長と相談しスクラムの導入を決定。
将来的にスクラムマスターになることを見越した上で導入推進担当となる。

マネジメント補佐の傍ら書籍やネットの情報、スクラムコーチへの質問を通して一通りの知識をつけたのち、チームメンバーとの勉強会を通して実施一歩手前までこぎつけるが、プロジェクトの状況変化により実施には至らなかった。

**後期: フロントエンド担当**

フロントサイドに所属し、TypeScript/Reactを使用して簡単な機能の実装を行った。
